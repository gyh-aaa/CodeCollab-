# 开发计划与学习路线

## 第 1 阶段：项目初始化

目标：搭建可持续迭代的前后端分离工程。

- 创建 monorepo 项目结构
- 创建 Vue 3 + TypeScript 前端工程
- 创建 Spring Boot 后端工程
- 配置 Docker Compose
- 设计数据库初版模型
- 配置统一响应、异常处理、跨域、接口文档

学习重点：

- Vite 工程结构
- Spring Boot 分层结构
- RESTful API 规范
- Docker Compose 基础

## 第 2 阶段：登录与权限

目标：完成真实后台系统的权限地基。

- 用户表、角色表、菜单表、权限表
- Spring Security + JWT
- 登录、登出、刷新 Token
- 当前用户信息接口
- 前端登录页
- 动态菜单
- 路由守卫
- 按钮权限指令

学习重点：

- JWT 原理
- RBAC 模型
- Spring Security 过滤器链
- Pinia 状态管理

## 第 3 阶段：组织与项目

目标：完成团队协作的业务主体。

- 组织 CRUD
- 组织成员管理
- 项目 CRUD
- 项目成员管理
- 项目归档
- 项目级权限判断

学习重点：

- 多对多关系建模
- MyBatis-Plus 分页查询
- 业务权限和系统权限的区别

## 第 4 阶段：任务核心功能

目标：完成项目中最核心的任务协作闭环。

- 任务 CRUD
- 任务状态流转
- 任务负责人
- 优先级
- 截止时间
- 标签
- 子任务
- 操作日志

学习重点：

- 复杂表单设计
- 状态流转建模
- 事务边界
- 查询条件封装

## 第 5 阶段：看板体验

目标：让项目具备真实协作工具的交互体验。

- 看板列
- 任务卡片
- 拖拽排序
- 拖拽修改状态
- 筛选与搜索

学习重点：

- 前端组件拆分
- 拖拽交互
- 乐观更新
- 后端排序字段设计

## 第 6 阶段：评论、附件、通知

目标：补齐协作场景里的沟通能力。

- 任务评论
- 评论回复
- MinIO 文件上传
- WebSocket 通知
- 消息中心
- 截止时间提醒

学习重点：

- 对象存储
- WebSocket
- 定时任务
- 消息已读/未读设计

## 第 7 阶段：数据统计与交付

目标：把项目打磨成可演示、可写简历的作品。

- 项目统计接口
- ECharts 图表
- 单元测试
- 接口文档
- Docker 部署
- README
- 简历项目描述

学习重点：

- 聚合查询
- 图表数据结构
- 测试策略
- 项目交付说明

## AI 辅助开发流程

每个模块按以下节奏推进：

1. 先让 AI 拆需求、页面、接口、表结构。
2. 再让 AI 生成初版代码。
3. 自己审查命名、异常、权限、事务和边界条件。
4. 让 AI 补测试、做重构、写文档。
5. 每完成一个模块提交一次 Git commit。

这套流程能训练你在真实团队里使用 AI 提升效率，而不是只复制代码。
